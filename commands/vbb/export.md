---
name: vbb:export
description: Export validated idea to GSD for execution
---
<objective>
Package the validated idea into a format that GSD can consume for execution planning.

This bridges the validation phase (VBB) with the execution phase (GSD).
</objective>

<process>
## Export to GSD

Compile all validation results into a concise brief that GSD can use.

### Prerequisites
- Must have completed at least Layer 1 (Quick) with GO result
- Should have completed Layer 2 (Validate) for better results

### Export Content
Create a summary that includes:

1. **Problem Statement**: What problem are we solving?
2. **Solution Overview**: What are we building?
3. **Validation Evidence**: What do we know to be true?
4. **Constraints**: What boundaries must we respect?
5. **Success Criteria**: What does "done" look like?

### Output Format

Create `.vbb/gsd-export.md` with:

```markdown
# GSD Brief: [Idea Name]

## Problem
[Clear problem statement]

## Proposed Solution
[High-level solution description]

## Validation Evidence
### What We Know
[Key validated assumptions]

### What We Assume
[Unvalidated but reasonable assumptions]

## Constraints
- Technical: [Known limitations]
- Resource: [Budget/time/skill constraints]
- Other: [Regulatory, legal, etc.]

## Success Criteria
[Measurable outcomes]

## Risk Notes
[Key risks GSD should plan for]

---
Generated by VBB - Validation Before Build
[Date]
```

After creating the export, display:
```
✓ Export created at .vbb/gsd-export.md

You can now run GSD commands:
  /gsd:new-project (paste the brief content when prompted)
```
</process>
